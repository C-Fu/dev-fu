Create two module scripts. Install follows curl-pipe binary download pattern. Remove follows RC-cleanup pattern from remove_rust.sh.

**modules/install_zoxide.sh:**
- Metadata header: `@name: Install zoxide`, `@params:`, `@platforms: linux, darwin`, `@version: 1.0.0`, `@deps: curl,wget`, `@timeout: 600`
- `set -eu` after header
- Copy `_maybe_sudo()` helper + FLU_PKG_MGR fallback verbatim
- Idempotent guard: `command -v zoxide` → print `zoxide already installed: $(zoxide --version)`, exit 0
- Install logic:
  - **darwin:** If brew available → `brew install zoxide`, else error
  - **linux (or unset FLU_OS):**
    - Detect arch: `uname -m` → map `x86_64` to `x86_64`, `aarch64` to `aarch64`, `armv7l` to `armv7`
    - Download from GitHub releases: `https://github.com/ajeetdsouza/zoxide/releases/latest/download/zoxide-${ARCH}-unknown-linux-musl.tar.gz`
    - Use curl or wget to download to `/tmp/zoxide.tar.gz`
    - Extract: `tar -xzf /tmp/zoxide.tar.gz -C /tmp zoxide`
    - Move: `_maybe_sudo mv /tmp/zoxide /usr/local/bin/zoxide`
    - `_maybe_sudo chmod +x /usr/local/bin/zoxide`
    - Clean up: `rm -f /tmp/zoxide.tar.gz`
  - **Other OS:** Error with "Visit https://github.com/ajeetdsouza/zoxide for manual install"
- After successful install, add shell integration:
  - For each RC file that exists (`$HOME/.bashrc`, `$HOME/.zshrc`):
    - Check if zoxide init line already present (`grep -q 'zoxide init' "$rcfile"`)
    - If not present, append: `printf '\neval "$(zoxide init bash)"\n' >> "$rcfile"` (use `bash` or `zsh` matching the RC file name)
  - For fish (`$HOME/.config/fish/config.fish`): append `zoxide init fish | source` if config exists
  - Print message: `printf 'zoxide installed. Restart your shell or source your RC file to activate.\n'`
- Success message: `printf 'zoxide installed successfully\n'`

**modules/remove_zoxide.sh:**
- Metadata header: `@name: Remove zoxide`, `@params:`, `@platforms: linux, darwin`, `@version: 1.0.0`, `@deps:`, `@timeout: 300`
- `set -eu` after header
- Copy `_maybe_sudo()` helper + FLU_PKG_MGR fallback
- Guard: `! command -v zoxide` → "zoxide is not installed", exit 0
- Remove:
  - **darwin:** `brew uninstall zoxide 2>/dev/null || true`
  - **linux:** `_maybe_sudo rm -f /usr/local/bin/zoxide`
- Clean RC files — for each of `$HOME/.bashrc`, `$HOME/.zshrc`, `$HOME/.config/fish/config.fish`:
  - `sed -i.bak '/zoxide init/d' "$rcfile" 2>/dev/null || true`
  - `rm -f "${rcfile}.bak" 2>/dev/null || true`
- Clean zoxide data: `rm -rf "$HOME/.local/share/zoxide" 2>/dev/null || true`
- Success message: `printf 'zoxide removed successfully. Shell integration lines cleaned from RC files.\n'`

Both scripts must be executable.

Create two module scripts. eza uses package manager where available with cargo as fallback. This is a hybrid pattern combining install_go.sh (package manager) and install_rust.sh (cargo fallback).

**modules/install_eza.sh:**
- Metadata header: `@name: Install eza`, `@params:`, `@platforms: linux, darwin`, `@version: 1.0.0`, `@deps: curl,wget`, `@timeout: 600`
- `set -eu` after header
- Copy `_maybe_sudo()` helper + FLU_PKG_MGR fallback verbatim
- Copy `_pkg_install()` and `_pkg_update()` dispatchers from install_go.sh verbatim
- Idempotent guard: `command -v eza` → print `eza already installed: $(eza --version)`, exit 0
- Install logic:
  - **darwin:** `brew install eza`
  - **linux:** Try package manager first, fall back to cargo:
    - If `FLU_PKG_MGR` is one of `apt`, `pacman`, `brew`:
      - `apt`: Add eza PPA/repository if needed, then `_pkg_install eza`
      - `pacman`: `_pkg_install eza`
      - `zypper`: `_pkg_install eza`
      - `dnf`: `_pkg_install eza` (available in EPEL, may need `dnf copr enable`
      - `apk`: Skip package manager (not available in Alpine repos), fall through to cargo
    - If package manager install fails or pkg mgr is `apk`:
      - Fall back to `cargo install eza` (requires Rust/cargo installed)
      - If `command -v cargo` fails: print "eza requires Rust/cargo for installation on this system. Install Rust first via the menu.", exit 1
  - Success message: `printf 'eza installed successfully\n'`

**modules/remove_eza.sh:**
- Metadata header: `@name: Remove eza`, `@params:`, `@platforms: linux, darwin`, `@version: 1.0.0`, `@deps:`, `@timeout: 300`
- `set -eu` after header
- Copy `_maybe_sudo()` helper + FLU_PKG_MGR fallback
- Copy `_pkg_remove()` dispatcher from remove_go.sh
- Guard: `! command -v eza` → "eza is not installed", exit 0
- Remove:
  - **darwin:** `brew uninstall eza 2>/dev/null || true`
  - **linux:**
    - Try package manager removal first: `_pkg_remove eza 2>/dev/null || true`
    - Also check if installed via cargo: `command -v cargo && cargo uninstall eza 2>/dev/null || true`
    - Clean up binary if still present: `_maybe_sudo rm -f /usr/local/bin/eza 2>/dev/null || true`
    - Clean up cargo binary: `rm -f "$HOME/.cargo/bin/eza" 2>/dev/null || true`
- Success message: `printf 'eza removed successfully\n'`

Both scripts must be executable.
  </action>

    - modules/install_eza.sh has platform dispatch for darwin (brew) vs linux (pkg mgr + cargo fallback)
