3. **`fust/src/platform.rs`** — Platform detection (per D-05):
   - `PlatformInfo` struct with fields:
     - `os: String` — "darwin" or "linux" (maps to FLU_OS)
     - `distro: String` — distro ID from /etc/os-release (maps to FLU_DISTRO)
     - `pkg_mgr: String` — detected package manager name (maps to FLU_PKG_MGR)
     - `arch: String` — CPU architecture (maps to FLU_ARCH)
     - `is_wsl: bool` — running under WSL (maps to FLU_IS_WSL)
     - `is_termux: bool` — running under Termux (maps to FLU_IS_TERMUX)
     - `is_root: bool` — running as root (maps to FLU_IS_ROOT)
   - `pub fn detect() -> Result<PlatformInfo>` function that produces IDENTICAL results to flu.sh's `flu_module_set_env()`:
     - **os**: use `std::env::consts::OS` — map "macos" → "darwin", keep "linux" as-is, default to "linux"
     - **distro**: read `/etc/os-release`, source it in a subprocess (`sh -c '. /etc/os-release && echo $ID'`), parse output. Fallback: "linux"
     - **pkg_mgr**: check command existence in priority order matching `_flu_detect_pkg_mgr` exactly: apt-get → apt, apk → apk, dnf → dnf, pacman → pacman, zypper → zypper, brew → brew, none → "unknown". Use `std::process::Command::new("sh").arg("-c").arg("command -v <tool>")` to match flu.sh's behavior.
     - **arch**: use `std::env::consts::ARCH` — map "x86_64" → "x86_64", "aarch64" → "aarch64", etc. This must match `uname -m` output.
     - **is_wsl**: read `/proc/version`, check for "microsoft" (case-insensitive). Return false if file doesn't exist (macOS).
     - **is_termux**: check env var `TERMUX_VERSION` is set OR `/data/data/com.termux` directory exists
     - **is_root**: run `id -u` and check if result is "0". Alternative: check env var `USER` == "root" as fallback.
   - `pub fn display(&self) -> String` method that formats platform info for display (matching flu.sh's startup info format):
     ```
     OS: linux | Distro: ubuntu | Package Manager: apt | Architecture: x86_64
     ```
   - `#[cfg(test)]` module with tests:
     - Test that `detect()` succeeds on the current platform (doesn't panic/error)
     - Test that `display()` produces non-empty output
     - Test that os is "linux" or "darwin"
     - Test that arch is non-empty

4. **`fust/src/main.rs`** — Entry point:
   ```rust
   mod cli;
   mod platform;
   mod menu;  // stub for task 2

   use clap::Parser;

   fn main() -> anyhow::Result<()> {
       let args = cli::Cli::parse();

       // Detect platform (always, matching flu.sh behavior)
       let platform = platform::detect()?;

       if args.list {
           // Delegate to menu module (implemented in task 2)
           let entries = menu::parse_menu_db();
           if args.json {
               menu::print_json(&entries);
           } else {
               menu::print_table(&entries);
           }
           return Ok(());
       }

       if args.install.is_some() || args.remove.is_some() {
           println!("Batch mode not yet implemented (coming in Phase 19)");
           std::process::exit(1);
       }

       // No args → TUI mode (Phase 16)
       println!("fust v{}", env!("CARGO_PKG_VERSION"));
       println!("{}", platform.display());
       println!("TUI mode not yet implemented (coming in Phase 16)");
       Ok(())
   }
   ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → fust | Platform detection reads /etc/os-release, /proc/version — system files that could be missing or malformed |
| embedded menu.db → parser | menu.db is embedded at compile time from repo — trusted source, but parser must handle malformed lines gracefully |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | Platform detection | accept | Reads from system files (/etc/os-release, /proc/version) that require root to modify; low-value target |
| T-15-02 | Denial of Service | menu.db parser | mitigate | Skip malformed lines silently (malformed = wrong number of pipe-delimited fields) — prevents panic on bad input |
| T-15-03 | Tampering | CLI args | accept | clap handles arg validation; no privileged operations in Phase 15 |
</threat_model>
