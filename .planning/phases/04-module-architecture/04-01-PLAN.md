Implement `flu_module_resolve_url(action_id)` (per D-04):
- Accept action_id as $1 (string like "install_python" from flu_menu_get_action)
- Base URL: `https://raw.githubusercontent.com/C-Fu/flu-modules/main/modules/`
- Override: Check `FLU_MODULES_BASE_URL` env var first; if set, use it as prefix instead
- Append `${action_id}.sh` to the base URL
- Print the full URL to stdout
- Return 0 on success

Implement `flu_module_fetch(action_id)` (per D-05, D-06, D-07, D-08):
- Call `flu_module_resolve_url "$1"` and capture the URL
- No temp file — pipe to stdout (caller reads the script content)
- Detect curl vs wget: if `command -v curl >/dev/null 2>&1`, use `curl -fsSL --connect-timeout 10 "$url"`; else use `wget -qO- --timeout=10 "$url" 2>/dev/null`
- Retry 3 times with 2-second delay on failure (per D-07). Implement retry loop WITHOUT using `local` or `seq` (POSIX sh): use a while loop with counter variable
- On persistent failure after all retries: print error to stderr with TUI_RED showing:
  - The failed URL
  - The HTTP exit code (curl exit code 22 = HTTP error, etc.)
  - Actionable message: "Check internet connection" for network errors, "Module not found — might be renamed" for 404
  - (per D-08)
- Use `set -o pipefail` before the curl/wget call to capture exit codes
- Return 0 on success, 1 on failure

All code must be POSIX sh: no `local`, no `[[ ]]`, no `echo -e` (use `printf`), no `${var:0:1}` substring, no bash arrays. Use global variables prefixed with `_flu_` for function scope.

Include ShellCheck pragma: `# shellcheck disable=SC2034,SC2154` at top of file (library constants and eval-assigned variables used by callers).

Append to `modules.sh` (do NOT overwrite task 1 code):

Implement `flu_module_parse_metadata()` (per D-01, D-02):
- Reads raw module script content from stdin (piped from flu_module_fetch)
- Extracts metadata from comment header using awk
- Header format: lines starting with `# @key: value` — terminated by first blank line or first non-comment line
- Parse these fields using awk regex:
  - `@name:` → display name (required)
  - `@params:` → parameter declarations (optional, empty string if absent)
  - `@platforms:` → comma-separated OS list (required)
  - `@version:` → version string (required)
  - `@deps:` → comma-separated dependency action IDs (optional, empty string if absent)
  - `@timeout:` → max execution seconds (optional, default "300")
- Output metadata fields to stdout in this exact order, one per line: name, params, platforms, version, deps, timeout
- On parse failure (missing required fields @name, @platforms, @version): print error to stderr with TUI_RED, return 1
- Validate platforms: check that the current OS is in the list. If not, print "Module not available for this platform" to stderr, return 1
- Use eval-assigned globals for parsed values: `_fmp_name`, `_fmp_params`, `_fmp_platforms`, `_fmp_version`, `_fmp_deps`, `_fmp_timeout`

Implement `_flu_parse_params(param_string)` (per D-03):
- Accept the @params string as $1 (e.g., "scope=radio:global,user;version=text")
- Return early if empty string
- Parse using awk with `-F';'`: split into individual param declarations
- For each declaration, split on `=` to get name and type_spec
- Split type_spec on `:` to get type and comma-separated choices
- Print to stdout one param per line in format: `index|name|type|choices`
  - Example output for "scope=radio:global,user;version=text":
    ```
    0|scope|radio|global,user
    1|version|text|
    ```
- Handles edge cases: params with no choices (text type), missing type (default to "text"), empty string
- If input contains no `=` separator: treat as error, return 1

Both functions must be POSIX sh compliant: no bashisms. Refactor any POSIX-incompatible patterns found during implementation.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Network (curl/wget) → modules.sh | Remote script content enters the system through stdin pipe |
| GitHub raw URL → resolver | URL string is constructed from user-supplied action_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | flu_module_fetch() — URL construction | mitigate | Action ID is not user-typed; it comes from menu.db DSL which is curated. Base URL defaults to project-owned repo. FLU_MODULES_BASE_URL override is documented as admin-only. |
| T-04-02 | Tampering | flu_module_fetch() — network fetch | accept | HTTPS enforced by GitHub raw URLs. SHA256 verification deferred to v2 (MODL-08). Module scripts are trusted because they come from project-owned repo. |
| T-04-03 | Information Disclosure | flu_module_parse_metadata() — stdout | accept | Metadata is non-sensitive (display names, versions). Platform info passed as env vars is already available via uname. |
| T-04-04 | Denial of Service | flu_module_fetch() — infinite retry | mitigate | Retry capped at 3 attempts (D-07). curl --connect-timeout 10 prevents hanging. |
</threat_model>
