```sh
flu_module_display_result() {
  _fdr_exit_code=$1
  _fdr_output=$2
  _fdr_name=$3
```

Implementation:

1. **Init TUI:** Call `tui_init` (terminal goes raw mode, signal traps set)

2. **Clear screen:** Call `clear_screen`

3. **Calculate box dimensions:**
   - Get terminal size: `_fdr_rows=$(tput lines 2>/dev/null || printf '24')`, `_fdr_cols=$(tput cols 2>/dev/null || printf '80')`
   - Box width: 70 columns (or cols-4 if terminal is narrower): `_fdr_box_w=70; [ "$_fdr_box_w" -gt $((_fdr_cols - 4)) ] && _fdr_box_w=$((_fdr_cols - 4))`
   - Box height: fill terminal minus 4 rows for margins: `_fdr_box_h=$((_fdr_rows - 4))`
   - Position: horizontally centered, vertically at row 2: `_fdr_x=$(( (_fdr_cols - _fdr_box_w) / 2 )); [ "$_fdr_x" -lt 1 ] && _fdr_x=1; _fdr_y=2`

4. **Render status banner (per D-12):**
   - Success (exit code 0): Set title to `✓ ${_fdr_name} — Complete` and color to `${TUI_GREEN}`
   - Failure (exit code != 0): Set title to `✗ ${_fdr_name} — Failed (exit: ${_fdr_exit_code})` and color to `${TUI_RED}`
   - Draw the box: `_tui_draw_box "$_fdr_x" "$_fdr_y" "$_fdr_box_w" "$_fdr_box_h" "$_fdr_title"`

5. **Render output content inside the box (per D-13):**
   - Inner width for content: `_fdr_inner=$((_fdr_box_w - 4))` (2 for borders + 2 padding)
   - Start rendering at row `_fdr_y + 2` (after title separator in _tui_draw_box)
   - For success: display the last meaningful lines of stdout (up to available space)
   - For failure: display stderr content (up to available space)
   - Split output into lines using `printf '%s' "$_fdr_output" | tr '\n' '|' | awk -F'|'` or iterate with while read
   - Each line: `move_cursor $_fdr_content_row $((_fdr_x + 2)); printf '%s' "$_fdr_line"` - truncate lines that exceed `_fdr_inner` width using `awk -v L="$_fdr_inner" '{print substr($0,1,L)}'`
   - Stop rendering when reaching `_fdr_y + _fdr_box_h - 4` (leaving room for footer and recovery hints)

6. **Failure: display recovery hints (per D-13):**
   - If exit code != 0, call `_flu_display_recovery_hints "$_fdr_exit_code"` at the bottom of the box (above footer row)
   - If no stderr output but non-zero exit: display "Module exited with code ${_fdr_exit_code} but produced no error output"

7. **Footer: "Press any key to continue":**
   - Render at `_fdr_y + _fdr_box_h - 1` (bottom row of box): `printf '%sPress any key to return to menu%s' "$TUI_DIM" "$TUI_RESET"`
   - Wait for keypress: call `_flu_wait_for_key`

8. **Restore TUI:** Call `tui_restore` (restores terminal and clears screen)

Functions to also implement in this task:

- Read a single keypress from /dev/tty to pause before returning to menu
- Call `_tui_read_key` (available from sourced tui.sh)
- Clear any pending input before returning

All code must be POSIX sh compliant. No bashisms.

**Part A: Implement `_flu_display_recovery_hints(exit_code)` (per D-13):**

This function is called from `flu_module_display_result` when the module fails. It renders recovery hints inside the result box based on common failure patterns.

Implementation:
```sh
_flu_display_recovery_hints() {
  _fdh_exit_code=$1
  _fdh_hint=""
```

Map exit codes and error patterns to actionable hints:

| Exit Code / Pattern | Recovery Hint |
|---------------------|---------------|
| 124 (timeout) | "The operation timed out after ${_fmp_timeout} seconds. Try again with a faster connection or check if the service is responsive." |
| 126 (not executable) | "The module script could not be executed. This may indicate a corrupted download. Try running again." |
| 127 (command not found) | "A required command was not found. Check that all dependencies are installed for this module." |
| 1 (generic error) | Check stderr content: if contains "curl" → network hint; if contains "Permission denied" → try with elevated privileges; if contains "not found" → dependency missing |
| Network-related (curl exit 6,7,22,28) | "Network error — unable to reach the server. Check your internet connection. If you're behind a proxy, set HTTP_PROXY and HTTPS_PROXY environment variables." |
| Any other | "Module exited with code ${_fdh_exit_code}. Check the output above for details. You can re-run this operation to try again." |

Render the hint inside the result box:
- Position: `_fdh_row=$((_fdr_y + _fdr_box_h - 4))` (two rows above bottom border, above "Press any key")
- Use TUI_YELLOW color for hints
- Word-wrap the hint text to fit within the box inner width (`_fdr_inner`)
- Call `move_cursor` then `printf '%s%s%s\n' "$TUI_YELLOW" "→ $_fdh_line" "$TUI_RESET"` for each wrapped line

**Part B: Wire result display into `flu_module_execute()`:**

In `flu_module_execute()`, the execution section needs to capture stdout and stderr to pass to the display function:

1. Capture stdout: Redirect stdout to a variable or temp file during execution.
   Since the module outputs to stdout (which the terminal displays), we need to capture it for the result modal. Use a temp file approach:
   ```sh
   _fmet_out="/tmp/flu_module_out_$$"
   _fmet_err="/tmp/flu_module_err_$$"
   # Execute module, capture output
   _flu_execute_with_timeout "$_timeout" "$_url" $_flu_module_args >"$_fmet_out" 2>"$_fmet_err"
   _flu_exit_code=$?
   # Read captured output
   _flu_module_output=$(cat "$_fmet_out" 2>/dev/null)
   _flu_module_stderr=$(cat "$_fmet_err" 2>/dev/null)
   rm -f "$_fmet_out" "$_fmet_err"
   ```

2. After execution, replace the old status display call with:
   ```sh
   # Display results (on success: stdout; on failure: stderr)
   if [ "$_flu_exit_code" -eq 0 ]; then

   else
     flu_module_display_result "$_flu_exit_code" "$_flu_module_stderr" "$_fmp_name"
   fi
   ```

3. Ensure the `flu_module_display_result` call is the LAST step in `flu_module_execute`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Module output → result display | Module stdout/stderr content is rendered directly in the terminal box — no HTML escaping needed, ANSI codes pass through (intentional for colored module output) |
| Temp file → filesystem | Output capture uses /tmp files — must be cleaned up on all exit paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Information Disclosure | flu_module_display_result() — output rendering | accept | Module output is displayed to the user who initiated execution. Module scripts are from the project's own repo. No sensitive data risk. |
| T-04-10 | Denial of Service | flu_module_display_result() — large output | mitigate | Output is truncated to fit available terminal height. Box inner width limits line length via awk substr. Terminal itself limits total display. |
| T-04-11 | Tampering | flu_module_execute() — temp file cleanup | mitigate | Temp files in /tmp/flu_module_* are cleaned via rm -f on every exit path. Files are owned by current user in private /tmp. |
</threat_model>
