# Phase 21 Plan 01: Build & Distribution Summary

Cross-compile fust Rust binary for 4 target platforms, set up GitHub Actions CI to build and release on tag push, and create a curl-pipe-bash installer script.

## What Was Built

### Task 1: Release Profile + Cross-Compilation Config
- Added `[profile.release]` to `fust/Cargo.toml` with `opt-level = "s"`, `lto = true`, `strip = true`, `panic = "abort"`, `codegen-units = 1`
- Created `fust/.cargo/config.toml` with linker config for `x86_64-unknown-linux-musl` and `aarch64-unknown-linux-musl`
- **Verified:** `x86_64-unknown-linux-musl` produces a **2.8MB statically linked binary** (well under 5MB target)
- All 114 existing tests pass

### Task 2: GitHub Actions CI/Release Workflow
- `.github/workflows/release.yml` with two workflow triggers:
  - **CI job** (push/PR to main): `cargo test` + `cargo clippy`
  - **Build + Release jobs** (tag push `v*`): 4-target build matrix
- Build matrix uses `cross` for Linux musl targets (Docker-based), `cargo` for macOS targets
- Strips binaries, packages as `.tar.gz`, generates SHA256 checksums
- Creates GitHub Release with `softprops/action-gh-release@v2` including all assets + merged `checksums.txt`

### Task 3: curl-pipe-bash Installer
- `install.sh` — 145 lines of POSIX sh (shellcheck-clean)
- Detects OS (Linux→musl, macOS→darwin) and arch (x86_64/aarch64) via `uname`
- Fetches latest release from GitHub API, or specific version via `FLU_VERSION` env var
- Downloads with curl (fallback wget), extracts, installs to `~/.local/bin/fust`
- PATH warning if install directory not in PATH
- Cleanup via `trap EXIT`

## Deviations from Plan
