<task type="auto">
  <name>task 1: Cargo release profile and cross-compilation config for static musl builds</name>
  <files>fust/Cargo.toml, fust/.cargo/config.toml</files>
  <read_first>fust/Cargo.toml</read_first>
  <action>
Modify `fust/Cargo.toml` to add a `[profile.release]` section with these exact settings:

```toml
[profile.release]
opt-level = "s"     # Optimize for size
lto = true          # Link-time optimization
strip = true        # Strip debug symbols
panic = "abort"     # No unwind tables — eliminates libgcc_s dependency for static musl
codegen-units = 1   # Better optimization at cost of compile time
```

This produces a fully static binary when targeting musl. The `panic = "abort"` is critical — it removes the need for libgcc_s (unwinding), which is the only dynamic dependency in the current build.

Create `fust/.cargo/config.toml` with cross-compilation target configuration:

```toml
# Cross-compilation targets for static builds
# Install with: rustup target add x86_64-unknown-linux-musl aarch64-unknown-linux-musl

[target.x86_64-unknown-linux-musl]
linker = "x86_64-linux-gnu-gcc"

[target.aarch64-unknown-linux-musl]
linker = "aarch64-linux-gnu-gcc"
```

NOTE: The macOS targets (x86_64-apple-darwin, aarch64-apple-darwin) do not need musl or special linker config — they compile natively on macOS runners in CI. They are NOT static but that is acceptable because macOS does not support static binaries; the binary will be self-contained (no non-system dependencies) thanks to rustls.

After modifying Cargo.toml, rebuild and verify:
```bash
cd fust && cargo build --release
strip -s target/release/fust -o /tmp/fust-stripped
ls -lh /tmp/fust-stripped
# Must be under 5MB
```

Install musl target and verify static build:
```bash
rustup target add x86_64-unknown-linux-musl
cargo build --release --target x86_64-unknown-linux-musl
file target/x86_64-unknown-linux-musl/release/fust
# Must show "statically linked"
strip -s target/x86_64-unknown-linux-musl/release/fust -o /tmp/fust-musl-stripped
ls -lh /tmp/fust-musl-stripped
# Must be under 5MB
```

If `x86_64-linux-gnu-gcc` is not installed, install it: `sudo apt-get install -y musl-tools`

<task type="auto">
  <name>task 2: GitHub Actions release workflow for 4-target build matrix</name>
  <files>.github/workflows/release.yml</files>
  <read_first>fust/Cargo.toml</read_first>
  <action>
Create `.github/workflows/release.yml` with a GitHub Actions workflow that:

1. **Triggers on:** tag push matching `v*` (e.g., v1.0.0)
2. **Build matrix:** 4 targets using `include` for target-specific settings:

| Target | Runner | Tool | Notes |
|--------|--------|------|-------|
| x86_64-unknown-linux-musl | ubuntu-latest | cross | Static Linux amd64 |
| aarch64-unknown-linux-musl | ubuntu-latest | cross | Static Linux arm64 |
| x86_64-apple-darwin | macos-latest | cargo | Native macOS amd64 |
| aarch64-apple-darwin | macos-latest | cargo | Native macOS arm64 |

3. **Steps per target:**
   - Checkout code
   - Install Rust toolchain (stable) with the matrix target
   - Install `cross` (cargo install cross --locked) for Linux targets only
   - Build: `cross build --release --target ${{ matrix.target }}` for Linux, `cargo build --release --target ${{ matrix.target }}` for macOS
   - Strip the binary (using appropriate strip command — `llvm-strip` for cross-compiled, `strip` for native)
   - Package: `tar -czf fust-${{ matrix.target }}.tar.gz -C target/${{ matrix.target }}/release fust`
   - Generate SHA256: `sha256sum fust-${{ matrix.target }}.tar.gz > fust-${{ matrix.target }}.sha256`
   - Upload artifacts

4. **Release job** (runs after all builds):
   - Download all artifacts
   - Create GitHub release using tag name as title
   - Upload all .tar.gz and .sha256 files as release assets
   - Generate checksums.txt with all SHA256 sums

5. **Artifact naming convention:**
   - `fust-x86_64-unknown-linux-musl.tar.gz`
   - `fust-aarch64-unknown-linux-musl.tar.gz`
   - `fust-x86_64-apple-darwin.tar.gz`
   - `fust-aarch64-apple-darwin.tar.gz`

Use `softprops/action-gh-release@v2` for release creation. Use `actions/upload-artifact@v4` and `actions/download-artifact@v4` for artifact passing.

Also add a CI workflow that runs `cargo test` and `cargo clippy` on push/PR to main branch. This can be in the same file or a separate `.github/workflows/ci.yml` — use a single file with two workflow triggers for simplicity.

IMPORTANT: The cross tool needs Docker. GitHub Actions ubuntu-latest runners have Docker pre-installed, so no extra setup needed.

<task type="auto">
  <name>task 3: curl-pipe-bash installer script with OS/arch detection</name>
  <files>install.sh</files>
  <read_first>.github/workflows/release.yml</read_first>
  <action>
Create `install.sh` at the repository root. This script must be:

1. **POSIX sh compatible** (no bashisms — `#!/bin/sh` shebang)
2. **curl-pipe-bash ready** — works when piped: `curl -fsSL <url> | sh`
3. **Zero dependencies** — only uses sh builtins, uname, curl/wget

The script must:

1. **Detect platform:**
   - OS: `uname -s` → "Darwin" maps to "apple-darwin", "Linux" maps to "unknown-linux-musl"
   - Arch: `uname -m` → "x86_64" maps to "x86_64", "aarch64"/"arm64" maps to "aarch64"
   - Combine into target triple: `{arch}-{vendor}-{os}`
   - Error with clear message if OS/arch is unsupported (e.g., Windows, i686)

2. **Resolve download URL:**
   - Default: fetch latest release from GitHub API: `https://api.github.com/repos/{owner}/{repo}/releases/latest`
   - Parse JSON to find the asset matching the target triple
   - Support `FLU_VERSION` env var for installing a specific version tag (e.g., `FLU_VERSION=v0.1.0`)
   - Support `FLU_REPO` env var for custom repo (default: `C-Fu/dev-fu`)

3. **Download:**
   - Prefer `curl -fsSL`, fallback to `wget -qO-` if curl not found
   - Download to a temp directory
   - Extract tar.gz: `tar -xzf`
   - Verify the binary exists after extraction

4. **Install:**
   - Default install location: `~/.local/bin/fust`
   - If `INSTALL_DIR` env var is set, use that
   - Create `~/.local/bin` directory if it doesn't exist
   - `chmod +x` the binary
   - Print success message with installed path
   - Print warning if `~/.local/bin` is not in PATH

5. **Cleanup:**
   - Remove temp directory on exit (trap EXIT)
   - Remove temp directory on error (trap ERR)

6. **Error handling:**
   - Print clear error messages for: unsupported OS, unsupported arch, download failure, extraction failure
   - Exit with non-zero code on any error

The script should be concise (under 150 lines) and well-commented. Follow the same POSIX style as flu.sh — no bashisms, shellcheck-clean.
  </action>

  </verify>

    - Downloads correct release binary from GitHub
