**Section 3: Box-drawing character auto-detection (per D-15)**

Call `_tui_detect_box_chars()` at file level so constants are available before any function runs.

**Section 4: TTY/terminal availability check**
Create a variable `_tui_use_tui=true`. Create function `_tui_check_tty()` that:
1. Tests `[ -c /dev/tty ]` — is /dev/tty a character device?
2. Tests `[ "${TERM:-}" != "dumb" ]` — is terminal not dumb?
3. Tests `[ -t 0 ]` — is stdin a terminal? (additional check)
4. If any check fails, set `_tui_use_tui=false`
Call `_tui_check_tty()` at file level.

**Section 5: Terminal init/restore (per D-06)**
Implement `tui_init()`:
1. If `_tui_use_tui` is false, return immediately (no TUI setup needed)
2. Save terminal state: `_tui_saved_stty=$(stty -g 2>/dev/null || true)`
3. Set raw mode: `stty -echo -icanon min 1 time 0 2>/dev/null || true`
4. Hide cursor: `printf '%s[?25l' "$ESC"`
5. Register signal traps — all four per CONCERNS.md (checklist.sh only has INT/TERM/HUP, missing QUIT):
   ```
   trap 'tui_restore; exit 130' INT
   trap 'tui_restore; exit 143' TERM
   trap 'tui_restore; exit 129' HUP
   trap 'tui_restore; exit 131' QUIT
   ```

Implement `tui_restore()`:
1. If `_tui_saved_stty` is non-empty: `stty "$_tui_saved_stty" 2>/dev/null || true`
2. Show cursor: `printf '%s[?25h' "$ESC"`
3. Clear all four traps: `trap - INT TERM HUP QUIT`
4. Set `_tui_saved_stty=''` to prevent double-restore

**Section 6: Rendering primitives**

- `clear_screen()` — `printf '%s[2J%s[H' "$ESC" "$ESC"`
- `_tui_clear_line()` — `printf '%s[2K%s[1G' "$ESC" "$ESC"` (clear current line, move to col 1)
- `_tui_printf_at(row, col, format, ...)` — move cursor then printf

All printf calls must use `%s` for the ESC variable — never embed raw escape bytes in format strings.

**Section 8: Shell-aware key reading (per D-09, D-10, D-11, D-12)**

Implement `_tui_read_byte()` — reads a single raw byte from /dev/tty:
- If `_tui_has_read_n` is true: `IFS= read -rsn1 _byte 2>/dev/null </dev/tty`
- If false: `_byte=$(dd bs=1 count=1 2>/dev/null </dev/tty || true)`
- IMPORTANT per D-10: dd usage is READ ONLY — only `dd bs=N count=N` from /dev/tty. No `of=`, `seek=`, or `conv=` flags.
- Print `_byte` to stdout. Return 0 on success, 1 on failure (EOF).

Implement `_tui_read_key()` — high-level key reader with escape sequence parsing:
1. Read first byte via `_tui_read_byte()`. Store in variable `key_byte`.
2. If read fails (EOF), return `TUI_KEY_UNKNOWN`.
3. Match the byte:
   - Newline (`$(printf '\n')` or `$(printf '\r')`): return `TUI_KEY_ENTER`
   - Space (`' '`): return `TUI_KEY_SPACE`
   - Tab (`$(printf '\t')`): return `TUI_KEY_TAB`
   - DEL (0x7f) or BS (0x08): return `TUI_KEY_BACKSPACE`
   - `q` or `Q`: return `TUI_KEY_Q`
   - `?`: return `TUI_KEY_HELP`
   - `j`: return `TUI_KEY_DOWN` (vi key per ENGN-01)
   - `k`: return `TUI_KEY_UP` (vi key per ENGN-01)
   - Digits 0-9: set global `_tui_digit_char` to the digit character, return `TUI_KEY_NUMBER`
   - ESC byte (`$(printf '\033')`): proceed to escape sequence parsing (step 4)
   - Home key (`G`): return `TUI_KEY_END` (vi-style End per success criteria)
   - `g`: return `TUI_KEY_HOME` (vi-style Home)
4. **Escape sequence parsing (per D-11):**
   - After reading ESC byte, temporarily set stty timeout for continuation byte read:
     ```
     _tui_key_timeout=${TUI_KEY_TIMEOUT:-10}  # deciseconds, default 100ms
     stty min 0 time "$_tui_key_timeout" 2>/dev/null || true
     ```
   - Read 1 continuation byte via `_tui_read_byte()`. If timeout (empty result), this is a bare Esc keypress — restore stty `min 1 time 0` and return `TUI_KEY_ESC`.
   - If continuation is `[` or `O`, this is an escape sequence — read one more byte:
     - After `[`: A=up, B=down, C=right, D=left, 5~=pgup, 6~=pgdn, H=home, F=end, 1~=home, 4~=end
     - After `O`: A=up, B=down, C=right, D=left, H=home, F=end
     - For `5~`, `6~`, `1~`, `4~`: need to read the `~` byte too (3-byte sequence after ESC+[)
   - After parsing, restore stty: `stty min 1 time 0 2>/dev/null || true`
   - If sequence not recognized, return `TUI_KEY_UNKNOWN`

**Section 9: Fallback numbered prompt (per D-07, D-08, ENGN-08)**

Implement `_tui_fallback_prompt(title, subtitle, item1, item2, ...)`:
1. Print blank line, then title (centered or left-aligned, plain text).
2. If subtitle provided, print it on next line.
3. Print separator line (`---`).
4. Loop through items (positional args starting at $3): print each as `  N) Label` with printf `%3d) %s`.
5. Print blank line.
6. Print prompt: `Enter number (or empty to cancel): `.
7. Read input: `IFS= read -r selection </dev/tty 2>/dev/null || read -r selection` (try /dev/tty first, then stdin).
8. If selection is empty, return exit code 1 (cancelled).
9. Validate selection is a positive integer. If not, print error and return exit code 1.
10. If selection is valid and in range (1 to item count): print the 0-based index to stdout, set `TUI_RESULT` global to the same value, return 0.
11. If out of range: print "Invalid selection" error, return exit code 1.

This function is used by `tui_select()` in Plan 02 when `_tui_use_tui` is false.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terminal input → script | Raw key bytes from /dev/tty are untrusted — crafted escape sequences could have unexpected lengths |
| Environment → script | TUI_KEY_TIMEOUT, TERM, LANG env vars are untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `_tui_read_key()` | mitigate | Escape sequence parser discards unrecognized sequences — bytes are never evaluated or interpolated |
| T-01-02 | Denial of Service | `_tui_fallback_prompt()` | accept | Reads single line of input — no buffer overflow risk in POSIX sh; terminal is local |
| T-01-03 | Information Disclosure | `tui_restore()` | mitigate | Terminal state is always restored even on signal — no lingering raw mode leaking keystrokes |
| T-01-04 | Elevation | `_tui_read_byte()` | mitigate | dd is read-only per D-10 — no `of=`, `seek=`, `conv=` flags; input is never passed to eval |
</threat_model>
