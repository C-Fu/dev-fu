**`parse_metadata(content: &str) -> Result<ModuleMetadata>`:**
- Parse the comment header from module script content
- Header terminates at first blank line or first non-comment line (matching modules.sh awk: `/^$/ { exit } !/^#/ { exit }`)
- Extract @key: value fields by line matching:
  - `^# @name: ` → name (required, error if missing)
  - `^# @params: ` → params string (optional, default "")
  - `^# @platforms: ` → platforms (required, comma-split + trim each, error if missing)
  - `^# @version: ` → version (required, error if missing)
  - `^# @deps: ` → deps (optional, default "")
  - `^# @timeout: ` → timeout (optional, parse as u64, default 300)
- Return error with descriptive message if any required field missing

**`validate_platform(metadata: &ModuleMetadata, current_os: &str) -> Result<()>`:**
- Check if `current_os` (e.g., "linux" or "darwin") is in `metadata.platforms`
- If not: return error "Module not available for this platform ({current_os})"

  - Assign incrementing index

**`execute_module(action_id: &str, platform: &PlatformInfo) -> Result<i32>`:**
This is the main orchestrator matching modules.sh `flu_module_execute()` (Section 9).

Pipeline steps:
1. **Fetch**: Call `crate::fetch::fetch_module(&FetchConfig::from_env(), action_id)` to get script content
2. **Parse metadata**: Call `crate::metadata::parse_metadata(&script_content)` to extract ModuleMetadata
3. **Validate platform**: Call `crate::metadata::validate_platform(&metadata, &platform.os)` — reject if OS mismatch
4. **Collect parameters**: If `metadata.params` is non-empty, call `collect_params(&metadata.params)` to get `Vec<(String, String)>` of (name, value) pairs. If user cancels (Esc), return error.
5. **Write script to temp file**: Write script content to `{TMPDIR}/flu_module_{pid}.sh` (or `/tmp/` if TMPDIR unset). Set executable permissions.
6. **Set environment variables**: Build env vars matching modules.sh `flu_module_set_env()`:
   - `FLU_OS` = platform.os
   - `FLU_DISTRO` = platform.distro
   - `FLU_PKG_MGR` = platform.pkg_mgr
   - `FLU_ARCH` = platform.arch
   - `FLU_IS_WSL` = if platform.is_wsl { "1" } else { "0" }
   - `FLU_IS_TERMUX` = if platform.is_termux { "1" } else { "0" }
   - `FLU_IS_ROOT` = if platform.is_root { "1" } else { "0" }
7. **Execute with timeout**: Call `execute_with_timeout(metadata.timeout, &script_path, &param_args, &env_vars)`
8. **Log execution**: Call `log_execution(action_id, &result, &metadata.version, duration_secs)`
9. **Print result**: Print success/failure to stderr matching modules.sh format:
   - Success: `  ✓ {name} — Complete`

10. **Cleanup**: Remove temp file

**`execute_with_timeout(timeout_secs: u64, script_path: &Path, args: &[String], env_vars: &[(String, String)]) -> Result<(i32, u64)>`:**
Matching modules.sh `_flu_execute_with_timeout()` (Section 8):
- Spawn: `std::process::Command::new("sh")` with args `["script_path", "--", ...param_args]`
- Set all env vars via `.env()` calls
- Use `child.kill()` after timeout via a watchdog thread:
  ```rust
  let child = Command::new("sh").arg(script_path).arg("--").args(args).envs(env_vars).spawn()?;
  let pid = child.id();
  let timeout_handle = std::thread::spawn(move || {
      std::thread::sleep(Duration::from_secs(timeout_secs));
      // Kill the child process if still running
      unsafe { libc::kill(pid as i32, libc::SIGKILL); }
  });
  let status = child.wait_with_output()?;
  // If timeout thread fired, the process was killed
  ```
  Actually, simpler approach — use `child.wait_timeout()` pattern:
  ```rust
  let start = Instant::now();
  let mut child = Command::new("sh").arg(script_path).arg("--").args(args)
      .envs(env_vars.iter().map(|(k,v)| (k.as_str(), v.as_str())))
      .spawn()?;
  
  // Wait with timeout using a loop
  let timeout = Duration::from_secs(timeout_secs);
  loop {
      match child.try_wait()? {
          Some(status) => {
              let duration = start.elapsed().as_secs();
              let code = status.code().unwrap_or(1);
              // Map SIGKILL (137) to timeout convention (124)
              let code = if code == 137 { 124 } else { code };
              return Ok((code, duration));
          }
          None => {
              if start.elapsed() > timeout {
                  child.kill()?;
                  let _ = child.wait();
                  let duration = start.elapsed().as_secs();
                  return Ok((124, duration));  // 124 = timeout convention
              }
              std::thread::sleep(Duration::from_millis(100));
          }
      }
  }
  ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fetch.rs → execute.rs | Fetched script content passed to execution engine (untrusted shell script) |
| execute.rs → subprocess | Shell script executed via `sh` with elevated privileges (may use sudo) |
| @params → TUI widgets | User input collected via TUI and passed as command-line arguments to scripts |
| execute.rs → log file | Execution results written to local filesystem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-07 | Elevation of Privilege | execute_with_timeout | mitigate | Module scripts run via `sh` (not `bash`), limiting shell expansion. FLU_* env vars are set by fust, not user input. Scripts that use sudo are module-level decisions, not fust's. |
| T-18-08 | Tampering | param_args | mitigate | Parameter values collected via TUI widgets (not parsed from user text input for radio/yesno). Text input values passed as `--key value` args, not interpolated into shell commands. |
| T-18-09 | Information Disclosure | log file | accept | Log contains action_id, result, version, duration — no secrets or PII. File at ~/.local/share/flu.sh/execution.log with default umask permissions. |
| T-18-10 | Denial of Service | execute_with_timeout | mitigate | Timeout enforcement (default 300s, max from @timeout metadata). Watchdog kills subprocess on timeout. Prevents runaway modules from blocking indefinitely. |
| T-18-11 | Spoofing | metadata parsing | mitigate | Metadata parsed from fetched script content that was already SHA256-verified by fetch.rs (18-01). Tampered scripts would fail SHA256 check before metadata is parsed. |
</threat_model>
