**1. TreeNode struct:**

```rust
pub struct TreeNode {
    pub label: String,          // display label (category, subcategory, or action label)
    pub path: Vec<String>,      // full path segments: ["Languages & Runtimes", "Go"]
    pub children: Vec<usize>,   // indexes into the flat nodes Vec
    pub action_id: Option<String>, // Some(action_id) for leaf items only
    pub depth: u8,              // 0=root, 1=category, 2=subcategory, 3=leaf
}
```

**2. MenuTree struct:**
```rust
#[derive(Debug, Clone)]
pub struct MenuTree {
    pub nodes: Vec<TreeNode>,   // flat storage (per D-01: flat list for data)
    pub root_children: Vec<usize>, // indexes of top-level category nodes
}
```

**3. `build_navigation_tree(entries: &[MenuEntry]) -> MenuTree`:**
- Iterate entries (already sorted by category, subcategory, label from parse_menu_db)
- Build unique category nodes (depth 1), subcategory nodes (depth 2), leaf nodes (depth 3)
- Use a HashMap<String, usize> to track node indexes by path key (e.g., "Languages & Runtimes" → idx)
- For each MenuEntry:
  - Create/find category node with path=[category]
  - Create/find subcategory node with path=[category, subcategory], add to category's children
  - Create leaf node with path=[category, subcategory, label], action_id=Some(entry.action_id.clone()), add to subcategory's children
- Return MenuTree { nodes, root_children: [indexes of category nodes] }

**4. Query methods on MenuTree (impl block):**
- `get_children(&self, path: &[String]) -> Vec<usize>` — if path is empty, return root_children; otherwise find node by path and return its children indexes
- `is_leaf(&self, node_idx: usize) -> bool` — returns true if node has no children (depth == 3 or children.is_empty())
- `get_breadcrumb(&self, path: &[String]) -> String` — format as "Main Menu > segment1 > segment2" with " > " separator (per D-02). Empty path → "Main Menu"
- `get_action_id(&self, node_idx: usize) -> Option<&str>` — return action_id if leaf
- `find_node_by_path(&self, path: &[String]) -> Option<usize>` — linear scan for matching path

**3. Main loop (mirrors menu.sh flu_menu_navigate logic):**
```
loop {
    let children = tree.get_children(&state.path);
    // clamp cursor to children.len()
    render(terminal, theme, tree, &state, &children);
    let key = input::read_key()?;
    // resolve go-to digits on non-number key (same pattern as select.rs)
    match key {
        Up/k    → cursor -= 1 (wrap to bottom)
        Down/j  → cursor += 1 (wrap to top)
        Home/g  → cursor = 0
        End/G   → cursor = children.len() - 1
        PgUp    → cursor -= page_size
        PgDn    → cursor += page_size
        Number  → go_digits accumulation (same logic as select.rs lines 76-93)
        Space   → per D-06: get child node, if leaf → queue.toggle(action_id)
        Enter   → per D-06:
                   if queue.count() > 0 → return Ok(Some(queue.to_vec()))
                   else: get selected child node
                     if is_leaf → return Ok(Some(vec![action_id]))
                     else → descend: path.push(child.label), cursor=0, scroll=0
        Esc/q   → per D-06:
                   if path is empty → return Ok(None) (quit from root)
                   else → go up: path.pop(), cursor=0, scroll=0
        Left/Backspace → if path not empty → path.pop(), cursor=0, scroll=0
        Help/?  → show_help = !show_help
        _       → ignore
    }
}
```

|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Spoofing | menu.db content | accept | Compile-time embedded via include_str! — cannot be spoofed at runtime. Menu.db is part of the binary. |
| T-17-02 | Tampering | Navigation tree | accept | Built from immutable embedded data at startup. No external mutation surface. |
| T-17-03 | Information Disclosure | Action IDs | accept | Action IDs are non-secret tool identifiers (install_go, install_rust). No sensitive data. |
| T-17-04 | Denial of Service | Menu rendering | accept | Finite menu.db (62 lines, 7 categories). No unbounded loops or recursive navigation. Tree has max depth 3. |
