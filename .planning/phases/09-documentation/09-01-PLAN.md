**Section 6 вҖ” Module Architecture:**
```markdown
## Module Architecture

flu.sh uses a remote on-demand module system. Each menu option maps to a standalone POSIX sh script under `modules/`. When flu.sh runs:

1. `tui.sh` provides ANSI terminal rendering primitives (cursor positioning, colors, keyboard input)
2. `menu.sh` parses `menu.db` (pipe-delimited menu DSL) and renders the interactive TUI
3. `modules.sh` handles remote script fetching from GitHub and execution

### How Modules Work

- **Local mode:** `git clone` and run вҖ” modules are sourced from disk, no network needed
- **Remote mode:** `curl-pipe-bash` вҖ” modules are fetched on-demand from GitHub raw URLs
- **Environment:** Modules use `FLU_PKG_MGR`, `FLU_OS`, `FLU_ARCH` for platform-aware installs
- **Safety:** All modules use `set -eu`, idempotent guards (`command -v`), and `_maybe_sudo()`

### Module Categories

| Category | Module Scripts | Description |
|----------|---------------|-------------|
| Languages | install_go.sh, install_rust.sh, install_python.sh, install_nvm_node.sh, install_bun.sh, install_php_laravel.sh (+ matching remove scripts) | Language toolchains and runtimes |
| Tools | install_docker.sh, install_tailscale.sh, install_yarn.sh, install_opencode_gsd.sh (+ matching remove scripts) | Developer tools and services |
| Shell | create_fancy_prompt.sh, create_fancy_prompt_blue.sh, install_avahi.sh (+ matching remove scripts) | Shell customization and network discovery |
| Diagnostics | status_check.sh, status_check_compare.sh, upgrade_all.sh | System status and batch operations |
| Settings | set_github_token.sh, configure_mouse_disable.sh, configure_mouse_enable.sh | Configuration helpers |
```

**Section 8 вҖ” Troubleshooting:**
```markdown
## Troubleshooting

### "command not found" after install
Some tools install to non-standard paths. Add to PATH:
`export PATH="$HOME/.cargo/bin:$HOME/.bun/bin:$HOME/.local/bin:$PATH"`

### Terminal not restored after exit
Press `Ctrl+C` or run `reset`. flu.sh has signal-safe cleanup via `_flu_cleanup_exit()`.

### Module fetch fails
flu.sh retries 3 times with 2s delays. For manual installs, clone the repo and run locally:
`git clone https://github.com/C-Fu/dev-fu.git && cd dev-fu && ./flu.sh`

### "No such file" on curl-pipe-bash
BusyBox and dash don't support process substitution (`<(curl ...)`). Use the alternate command:
`curl -fsSL https://raw.githubusercontent.com/C-Fu/dev-fu/refs/heads/flu.sh/flu.sh -o /tmp/flu.sh && sh /tmp/flu.sh`

### Permission denied
`chmod +x flu.sh`
```
