### Function 1: `flu_batch_run <action_ids> <flags>`
Implements non-interactive batch module execution per D-05 through D-09.

**Parameters:**
- `$1` = comma-separated action_ids (e.g., "install_go,install_rust,remove_python")
- `$2` = flags string — "yes" if `--yes` was passed, empty otherwise

**Logic:**
1. Call `flu_module_set_env` to set platform context (same as interactive mode).
2. Split `$1` on commas into individual action_ids using IFS.
3. For each action_id:
   a. Print status line: `▶ <action_id>` (using TUI_BOLD if TTY, plain if not)
   b. Call `flu_module_fetch "$action_id"` to a temp file — on failure: print `✗ <action_id> — Fetch failed`, increment `_batch_fail`, continue to next.
   c. Call `flu_module_parse_metadata` on the temp file — on failure: print `✗ <action_id> — Metadata parse error`, increment `_batch_fail`, continue to next.
   d. Check `_fmp_params`: if non-empty AND `$2` contains "yes": print `✗ <action_id> — Requires parameters, use interactive mode`, increment `_batch_fail`, continue. If non-empty AND `$2` does NOT contain "yes": this should not happen (prompted mode is interactive), but skip with warning.
   e. Platform check: iterate `_fmp_platforms` comma-separated list, compare to `$FLU_OS`. If no match: print `✗ <action_id> — Not available for this platform`, increment `_batch_fail`, continue.
   f. Execute module in subshell with timeout: `_flu_execute_with_timeout "$_fmp_timeout" "$_tmp_file"`. Capture exit code.
   g. Call `_flu_log_execution` with the same args as `flu_module_execute` does (action_id, operation, result, version, duration).
   h. If exit code 0: print `✓ <action_id> — Complete` (strip ANSI if not TTY via `[ -t 1 ]`). Increment `_batch_ok`.
   i. If exit code non-0: print `✗ <action_id> — Failed (exit N)`. Increment `_batch_fail`.
   j. Clean up temp file.
4. After all modules processed, print blank line, then summary: `N succeeded, M failed` using `_batch_ok` and `_batch_fail` counters.
5. Return 0 if `_batch_fail` is 0, return 1 if any failures (per D-08).
6. Strip ANSI escape codes from ALL output when `[ -t 1 ]` is false (per D-09). Implement a helper `_flu_strip_ansi()` that pipes through `sed 's/\x1b\[[0-9;]*m//g'`.

**Variable naming:** Use `_batch_*` prefix for all locals to avoid collision with `_fme_*` from `flu_module_execute`.

### Function 2: `flu_batch_list <json_flag>`
Implements `--list` output per D-03.

**Parameters:**
- `$1` = "json" for `--list --json`, empty for plain text table

**Logic (plain text):**
1. Read `menu.db` line by line.
2. Skip comment lines (starting with `#`) and empty lines.
3. Parse each line: `Category|Subcategory|Label|action_id` using `IFS='|'`.
4. Check installed status: `command -v` doesn't work for most tools — instead, just show the table without installed status for now (installed detection is complex and varies per tool). Print: `Category  Name  Action_ID` in columnar format using `printf` with fixed widths.
5. Sort by category then name.

**Logic (JSON):**
1. Print opening `[` bracket.
2. For each menu.db entry, print JSON object: `{"category":"...","subcategory":"...","name":"...","action_id":"..."}`.
3. Print closing `]` bracket.
4. One object per line, comma-separated (trailing comma on last item removed).

```sh
_flu_cli_mode=false
_flu_cli_install=''
_flu_cli_remove=''
_flu_cli_list=false
_flu_cli_yes=false
_flu_cli_json=false

while [ $# -gt 0 ]; do
    case "$1" in
        --install)
            [ -z "${2:-}" ] && { printf 'Error: --install requires a comma-separated list of action IDs\n' >&2; exit 2; }
            _flu_cli_install="$2"
            shift 2
            ;;
        --remove)
            [ -z "${2:-}" ] && { printf 'Error: --remove requires a comma-separated list of action IDs\n' >&2; exit 2; }
            _flu_cli_remove="$2"
            shift 2
            ;;
        --list)
            _flu_cli_list=true
            shift
            ;;
        --yes)
            _flu_cli_yes=true
            shift
            ;;
        --json)
            _flu_cli_json=true
            shift
            ;;
        --help|-h)
            printf 'Usage: flu.sh [OPTIONS]\n\n'
            printf 'Options:\n'
            printf '  --install <ids>  Install modules (comma-separated action IDs)\n'
            printf '  --remove <ids>   Remove modules (comma-separated action IDs)\n'
            printf '  --list           List available modules\n'
            printf '  --yes            Skip confirmations (batch mode)\n'
            printf '  --json           JSON output (with --list)\n'
            printf '  --help           Show this help message\n'
            exit 0
            ;;
        *)
            printf 'Error: Unknown option: %s\n' "$1" >&2
            printf 'Try: flu.sh --help\n' >&2
            exit 2
            ;;
    esac
done
```
