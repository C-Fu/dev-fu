  <action>
    Add execution logging to `flu_module_execute()` in `modules.sh`. Every module execution (success or failure) is recorded to a TSV log file.

    **Log file location (D-11):** `$FLU_DATA_DIR/execution.log` where `FLU_DATA_DIR` defaults to `${XDG_DATA_HOME:-$HOME/.local/share}/flu.sh`. The `FLU_DATA_DIR` constant was added by Plan 01 near the top of modules.sh.

    **Log format (D-10):** Tab-separated values, one line per execution:
    ```
    timestamp\taction_id\toperation\tresult\tversion\tduration_seconds
    ```
    - `timestamp`: ISO 8601 format (`date +%Y-%m-%dT%H:%M:%S%z` or `date -Iseconds` if available)
    - `action_id`: the action identifier passed to flu_module_execute (e.g., `install_go`)
    - `operation`: extract from action_id — if starts with `install_` then "install", if starts with `remove_` then "remove", if starts with `create_` then "create", if starts with `configure_` then "configure", if starts with `set_` then "set", if starts with `status_` then "status", if starts with `upgrade_` then "upgrade", else "other"
    - `result`: "success" if exit code 0, "fail" otherwise
    - `version`: the `_fmp_version` metadata value (e.g., "1.0.0"), or "-" if not available
    - `duration_seconds`: wall-clock time of execution in seconds (integer), measured from just before `_flu_execute_with_timeout` call to just after

    **Implementation approach:**

    1. Add a logging helper function `_flu_log_execution()` near the other helper functions (before `flu_module_execute`):
       ```sh
       # _flu_log_execution <action_id> <operation> <result> <version> <duration>
       # Appends execution record to TSV log file per D-10/D-11/D-12.
       _flu_log_execution() {
         _fle_action="$1"
         _fle_operation="$2"
         _fle_result="$3"
         _fle_version="${4:--}"
         _fle_duration="${5:--}"
         _fle_ts=$(date +%Y-%m-%dT%H:%M:%S%z 2>/dev/null || date +%Y-%m-%dT%H:%M:%S)
         _fle_logfile="${FLU_DATA_DIR}/execution.log"
         
         # Ensure log directory exists
         mkdir -p "${FLU_DATA_DIR}" 2>/dev/null || true
         
         # Write header if log file is new/empty
         if [ ! -f "$_fle_logfile" ] || [ ! -s "$_fle_logfile" ]; then
           printf 'timestamp\taction_id\toperation\tresult\tversion\tduration_seconds\n' > "$_fle_logfile"
         fi
         
         # Append log entry
         printf '%s\t%s\t%s\t%s\t%s\t%s\n' \
           "$_fle_ts" "$_fle_action" "$_fle_operation" "$_fle_result" "$_fle_version" "$_fle_duration" \
           >> "$_fle_logfile"
         
         unset _fle_action _fle_operation _fle_result _fle_version _fle_duration _fle_ts _fle_logfile
       }
       ```

    2. Add an operation extraction helper `_flu_classify_operation()`:
       ```sh
       # _flu_classify_operation <action_id>
       # Extracts operation type from action_id prefix per D-10.
       _flu_classify_operation() {
         case "$1" in
           install_*)   printf 'install' ;;
           remove_*)    printf 'remove' ;;
           create_*)    printf 'create' ;;
           configure_*) printf 'configure' ;;
           set_*)       printf 'set' ;;
           status_*)    printf 'status' ;;
           upgrade_*)   printf 'upgrade' ;;
           *)           printf 'other' ;;
         esac
       }
       ```

    3. Modify `flu_module_execute()` to:
       a. Record start time before Step 6 (execution):
          ```sh
          _fme_start=$(date +%s)
          ```
       b. After Step 6 (after `_flu_execute_with_timeout` returns), calculate duration:
          ```sh
          _fme_end=$(date +%s)
          _fme_duration=$((_fme_end - _fme_start))
          ```
       c. Classify the operation:
          ```sh
          _fme_op=$(_flu_classify_operation "$_fme_action")
          ```
       d. Determine result:
          ```sh
          if [ "$_fme_exit_code" -eq 0 ]; then
            _fme_result="success"
          else
            _fme_result="fail"
          fi
          ```
       e. Call logging helper:
          ```sh
          _flu_log_execution "$_fme_action" "$_fme_op" "$_fme_result" "${_fmp_version:-}" "$_fme_duration"
          ```
       f. Clean up new variables in the cleanup section:
          ```sh
          unset _fme_start _fme_end _fme_duration _fme_op _fme_result
          ```

    **Important:** The logging must happen BEFORE the cleanup of temp files and BEFORE the "Press any key" prompt (so the log is written even if the user Ctrl+C during the prompt). Insert the logging call between Step 6 (execution result) and Step 7 (display/wait).

    **Append-only (D-12):** Never truncate or rotate the log. Each execution appends one line. No size check needed — one line per execution is trivially small.

    **Error handling:** If log directory creation fails (e.g., read-only filesystem), silently skip logging — do not block module execution. Wrap the `_flu_log_execution` call:
    ```sh
    _flu_log_execution "$_fme_action" "$_fme_op" "$_fme_result" "${_fmp_version:-}" "$_fme_duration" 2>/dev/null || true
    ```

    **Edge cases:**
    - If module fails at fetch/parse/platform stage (before execution), log with result "fail" and duration 0
    - If metadata parsing fails, version field is "-" (unknown)
    - Log for successful fetch-but-failed-execution records actual duration
  </action>
