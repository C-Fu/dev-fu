    # Fetches MANIFEST.sha256 from the same base URL as modules.
    # Outputs manifest content to stdout on success.
    # Returns 0 on success, 1 on failure (soft-fail per D-03).
    _flu_fetch_manifest() {
      _ffm_base="${FLU_MODULES_BASE_URL:-https://raw.githubusercontent.com/C-Fu/dev-fu/main/flu-sh/modules/}"
      _ffm_url="${_ffm_base}MANIFEST.sha256"

    }

    **Rewrite `flu_module_fetch()` with all three features:**

    The new fetch flow per D-01 through D-09:
    1. Check local file first (existing behavior, unchanged)
    2. Check cache: if `~/.cache/flu.sh/<action_id>` exists and mtime < now - TTL → serve from cache, print "Using cached <action_id>" to stderr

    - Cache directory: `$FLU_CACHE_DIR` (default `~/.cache/flu.sh/`)

    - Cache hit: file exists AND `(current_epoch - file_mtime) < $FLU_CACHE_TTL`
    - On cache hit: `cat` the cached file to stdout, return 0
    - On cache miss/expired: proceed to fetch
    - After successful fetch + verify: write to cache file, creating directory if needed (`mkdir -p "$FLU_CACHE_DIR"`)
