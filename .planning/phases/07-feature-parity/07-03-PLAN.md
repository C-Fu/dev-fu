**Key challenge:** Fancy Prompt scripts must convert fu.sh's bash-specific prompt templates (`echo -e`, `PS1=`, `$'...'` ANSI-C quoting, `${debian_chroot:+...}`) to pure POSIX sh (`printf`, variable assignment without bashisms). Use `printf` for color output and `\033[...m` escape sequences.

**Per-tool specifications:**

### create_fancy_prompt.sh
- fu.sh reference: `create_fancy_prompt()` (lines 1137-1159), prompt template (lines 689-982)
- Operation: Append a PS1 prompt definition to ~/.bashrc
- Platforms: `linux, darwin`
- **POSIX adaptation:** Convert the Purple-Pink starburst prompt from bash to POSIX sh. The fu.sh version uses:
  - `PS1='${debian_chroot:+($debian_chroot)}...'` — bash parameter expansion
  - `$'...'` ANSI-C quoting for escape sequences
  - `echo -e` for color output
  - Convert to: `printf` with octal escapes, simplified PS1 using `[` `\]` bash prompt escapes (which are bash-specific but the prompt goes INTO .bashrc, so bashisms in the PS1 string are actually OK — the prompt is written to .bashrc which bash reads)
- Actually, since the prompt string goes INTO ~/.bashrc (which is sourced by bash), the PS1 content can use bash syntax. The MODULE script itself must be POSIX sh, but the CONTENT it writes to .bashrc can be bash syntax.
- Module logic: `printf '...PS1=...' >> "$HOME/.bashrc"`, then `printf 'source ~/.bashrc to activate'`
- Idempotent: check if `~/.bashrc` already contains the dev-fu prompt marker comment
- Add a marker comment: `# dev-fu purple-pink prompt` so remove_fancy_prompt can find and remove it
- `@timeout: 60` (file write only, fast)

- Set file permissions: `chmod 600` on token file

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub token storage | Token is written to ~/.config/gh/hosts.yml or ~/.github_token — file permissions must restrict access |
| npm global install | install_opencode_gsd uses npm install -g — depends on npm registry integrity |
| curl-pipe URLs | status_check_compare uses external APIs (GitHub, npm, PyPI) — network dependency |
| Shell config files | Fancy prompt scripts write to ~/.bashrc — could conflict with existing customizations |
| Terminal escape codes | Mouse reporting scripts send raw escape sequences to terminal — safe but could confuse some terminals |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Information Disclosure | set_github_token.sh — token file permissions | mitigate | `chmod 600` on token file. Only readable by owner. Print warning if token file already exists with wrong permissions. |
| T-07-08 | Tampering | create_fancy_prompt.sh — ~/.bashrc modification | mitigate | Use marker comments (`# dev-fu purple-pink prompt start/end`) to isolate prompt block. Remove script deletes only marked section. Backup original .bashrc before modification? ACCEPT — no backup, markers are sufficient. |
| T-07-09 | Elevation of Privilege | upgrade_all.sh — package manager commands via _maybe_sudo | mitigate | _maybe_sudo pattern (only sudo when not root and sudo available). User sees sudo prompt for each escalated command. |
| T-07-10 | Denial of Service | status_check_compare.sh — external API calls | accept | API calls may timeout or fail — script handles failures gracefully with "latest check not available" messages. @timeout set to 300s. |
</threat_model>
