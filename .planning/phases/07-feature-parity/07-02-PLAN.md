<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| curl-pipe install scripts | Rust (rustup), Bun (bun.sh), NVM (nvm-sh), Tailscale (tailscale.com) all download and pipe to sh — trusting external URLs |
| npm global install | yarn uses `npm install -g` — depends on npm being available and not compromised |
| sudo usage | Docker, package manager commands use sudo — privilege escalation boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03 | Spoofing | curl-pipe URLs (rustup.rs, bun.sh, nvm-sh, tailscale.com) | mitigate | Use HTTPS URLs only. curl -fsSL with TLS verification (default). The flu_module_fetch pipeline already uses https for module fetching. |
| T-07-04 | Tampering | Downloaded install scripts | accept | curl-pipe is the standard install method for these tools. SHA256 verification is deferred to v2 per REQUIREMENTS.md. |
| T-07-05 | Elevation of Privilege | sudo in _maybe_sudo() | mitigate | _maybe_sudo only escalates when not already root. Docker usermod adds to docker group (equivalent to root). All sudo operations are explicit in script output. |
| T-07-06 | Denial of Service | Package manager operations | accept | Package manager installs can be slow/network-dependent. @timeout values set appropriately (300s pkg, 600s curl-pipe). |
</threat_model>
