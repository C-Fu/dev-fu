<objective>
Make `fu.sh` OK for Alpine by ensuring `bash` is installed before OpenCode and any npm-dependent install runs.

Purpose: Alpine/musl ships no `bash` by default (only `ash`/busybox). Several downstream install paths require bash: the NVM installer (`bash /tmp/nvm-install.sh`), the Bun installer (`bash /tmp/bun-install.sh`), and npm native-module builds (node-gyp / postinstall scripts for packages like `better-sqlite3` and `node-pty` used by OpenChamber — see `npm_error.log`). On a minimal Alpine these fail. Adding a bash prerequisite guard before the opencode/npm install functions makes the workflow self-bootstrapping on Alpine.

Output: One new helper `_ensure_alpine_bash()` in `fu-sh/fu.sh`, wired into the four npm-dependent install functions, plus a contract test.
</objective>

    esac
}

_ensure_alpine_bash() {
    # Alpine/musl ships no bash by default (only ash/busybox). Downstream
    # installs invoked from this script need bash: the NVM and Bun installer
    # scripts (bash /tmp/*-install.sh) and npm native-module builds (node-gyp

}
